# Create the multi-signature wallet contract

The multi-signature wallet needs to do the following:

* Parse multi-signatures sent to it and validate them
* Store information about valid signers for validation
* Handle external messages
* Transfer default tokens and/or custom currencies

:::info

To validate signatures, the multi-signature wallet needs to use the `Nil.validateSignature(pubkey, hash, authData)` function. The function can only accept `authData` that is 256 bytes or less. As a signatures takes up 65 bytes, the wallet can only support three valid signers at maximum.

:::


The contract stores the signers' public keys inside the `pubkeys` array. The constructor checks for the length of the passed `_publicKeys` array and assigns `pubkeys` to this array as long as its length is correct.

## Parsing multi-signatures and validating them

To handle external messages, the contract uses the `verifyExternal()` function:

```solidity showLineNumbers file=../../../tests/MultiSigWallet.sol start=startVerifyExternal end=endVerifyExternal
```

The function iterates over all signatures passed in the `signature` calldata using the `offset` counter. Each iteration calls the `parseSignature()` function:

```solidity showLineNumbers file=../../../tests/MultiSigWallet.sol start=startParseSignature end=endParseSignature
```

The function reconstructs an individual signature when given 65 bytes from the given offset (`_pos`). It does so by extracting r, s, and v components and using `mstore` and `mstore8`.
